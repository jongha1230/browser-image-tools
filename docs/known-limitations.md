# Browser Support And Known Limitations

## 브라우저 지원 범위

- 2026년 3월 15일 기준 수동 검증 범위는 production demo의 Chromium 캡처 흐름입니다.
- 캡처 뷰포트는 데스크톱 `1440x900`, 모바일 `390x844`를 사용했습니다.
- 현재 구현은 Canvas, Blob, `URL.createObjectURL`, Web Worker, 다운로드 링크를 지원하는 최신 브라우저를 전제로 합니다.
- Web Worker가 초기화되지 않거나 지원되지 않으면 메인 스레드 폴백 경로로 같은 처리 파이프라인을 실행합니다.

## 현재 제한 사항

- 입력과 출력은 JPEG, PNG, WebP만 지원합니다.
- 모든 처리는 브라우저 메모리 안에서 이뤄지므로 매우 큰 파일이나 대량 배치에서는 느려지거나 탭 메모리 사용량이 커질 수 있습니다.
- 브라우저가 특정 형식 재인코딩을 지원하지 않으면 일부 출력 형식은 실패할 수 있습니다.
- EXIF 제거는 재저장 방식이라 메타데이터 정리에 도움이 되지만, 이미지 안에 보이는 민감 정보까지 숨기지 않습니다.
- 분석 스크립트와 광고 스크립트는 아직 연결하지 않았습니다.

## 범위 밖 항목

- 백엔드 업로드
- 계정, 인증, 데이터베이스
- 클라우드 저장 또는 동기화
- PDF, HEIC, RAW, 비디오 처리
- 엔드유저 대상 AI 기능

## 후속 개선 메모

- 대용량 배치 처리 시 메모리 사용량과 진행률 체감 개선
- 도구별 프리셋과 입력 검증 문구 세분화
- 실제 광고 운영 정보가 준비된 뒤 정책 고지와 관련 기술 파일 보강
- 브라우저별 수동 QA 범위를 Chromium 밖으로 확장

증빙 캡처와 아키텍처 요약은 [docs/proof-assets.md](./proof-assets.md)에서 함께 확인할 수 있습니다.
